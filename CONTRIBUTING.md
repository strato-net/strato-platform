# Contributing Guidelines

There are many ways to contribute to STRATO.

Please read our disclaimer first: DISCLAIMER.md

## Ways to Contribute

### Opening an issue

You can open an issue to suggest a feature or report a minor bug. But before
you do, be sure to search through the existing open and closed issues to
ensure you are not duplicating a suggestion that already exists. If it is
already there, please consider posting a comment on it first.

When requesting a new feature, include as many details as you can, especially
around the use cases that motivate it. Features are prioritized according to
the impact they may have on the ecosystem, so we appreciate information
showing that the impact could be high.

### Submitting a pull request

Well-structured contributions that address real issues are welcome. When in
doubt, reach out to us first to discuss your planned changes. To prevent
abuses (e.g., metric-farming), we reserve the right to block users who
repeatedly submit low-effort or trivial content without meaningful
improvements, as well as disruptive actions like excessively trying to get the
attention of the team, on GitHub or elsewhere.

If you would like to contribute code or documentation you may do so by forking
the repository and submitting a pull request.

If you're planning to take on a task, let us know first so we aren't
duplicating the effort.

### DCO

All commits must include a DCO sign-off line using your legal name.
Git can add this automatically with the -s or --signoff option, as long as
your user.name and user.email are set correctly.

TL;DR:

Ensure every commit has a sign-off.

git config user.name "FIRST_NAME LAST_NAME" git config user.email
"MY_NAME@example.com"

git commit -s -m "Your message"

If you use the GitHub web UI for commits, make sure your email address is
public in your GitHub profile to avoid the XYZ@users.noreply.github.com
placeholder.

Optional: set a git alias for convenience.
